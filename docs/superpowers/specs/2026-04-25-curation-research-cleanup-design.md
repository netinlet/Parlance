## Phase A — Move docs to vault

For each item in the Promote section above:

1. Ensure target vault directory exists (`mkdir -p`).
2. Run a `diff -q` against any existing vault file at the candidate target
   (currently only the `parlance-workspace-design` spec has a possible vault
   neighbour). If identical, skip the move and just delete the repo copy.
3. Move the file: `mv <repo path> <vault path>`. Use `mv` so no copy is left
   in the working tree. For staged files, run
   `git restore --staged <repo path>` first so `mv` doesn't leave the index
   in a half-staged state.
4. After the file moves into the vault, the repo working tree no longer has
   it; the next `git status` reflects deletion (for previously-tracked files)
   or simply absence (for untracked files).

For each item in the Drop section: `rm` the file (or
`git restore --staged && rm` if staged).

For each item in the Already-in-vault section: `git restore --staged && rm`.
No vault write.

**End-state of Phase A:** working tree should contain only the two untracked
python tools (`tools/generate-curation-views.py`,
`tools/validate-curation-sets.py`) and whatever existing tracked files were
modified by this branch and not part of the cleanup (the `M` set vs `main`).
The `A` and untracked sets should be empty except for the two python tools.
